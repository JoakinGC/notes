useContext

useContext en React

¿Qué es useContext?

useContext es un hook de React que te permite acceder al valor de un contexto dentro de un componente funcional, sin necesidad de usar un <Context.Consumer>.

const valor = useContext(MiContexto);

🔍 ¿Para qué sirve?

En una aplicación React, los datos como el tema visual, idioma, autenticación, usuario, etc., pueden ser compartidos entre muchos componentes. Usar useContext junto con React.createContext permite:

  • Evitar el prop drilling (pasar props innecesarias a través de muchos niveles).
  • Centralizar información global que puede cambiar.

🧱 ¿Cómo se usa?

1. Crear el contexto:

import { createContext } from "react";

export const TemaContext = createContext("claro");

2. Proveer el contexto en el árbol de componentes:

import { TemaContext } from "./TemaContext";

function App() {
  return (
    <TemaContext.Provider value="oscuro">
      <MiComponente />
    </TemaContext.Provider>
  );
}

3. Usarlo en un componente:

import { useContext } from "react";
import { TemaContext } from "./TemaContext";

function MiComponente() {
  const tema = useContext(TemaContext);

  return <div className={tema}>Modo actual: {tema}</div>;
}

🔁 Reactividad

Si el value del Provider cambia, todos los componentes que usen useContext se vuelven a renderizar automáticamente.

⚠️ Consideraciones importantes

  • useContext solo funciona con contextos creados con React.createContext.
  • Si el contexto cambia muy seguido o contiene objetos o funciones sin memorizar, puede causar renderizados innecesarios.
  • Para valores complejos (objetos, funciones), considera usar useMemo o useCallback junto al Provider.

Ejemplo práctico: Autenticación

// authContext.js
export const AuthContext = createContext(null);

// App.jsx
<AuthContext.Provider value={{ user: "joaquin", loggedIn: true }}>
  <Dashboard />
</AuthContext.Provider>

// Dashboard.jsx
const { user, loggedIn } = useContext(AuthContext);
