Canalización de machine learning

Proceso o flujo donde se muestra como se crea ML.

  1. Formulación del problema.
    Expresar el problema empresarial, del que usted o equipo pueden beneficiar de ML

  2. Planteamiento del problema:

Cuando ya se tiene claro y información del problema se debe evaluar si es mas útil usar ML o un solución tradicional al problema. Si tambien si se trata de un ML supervisado o sin supervisar. Si cuenta con datos catalogados para entrenar el modelo. Debes hacerte muchas preguntas y validar si es útil crear ML y si cuenta con datos para entrenarlo.

Ejemplo: Tenemos un problema de membresía en donde los clientes están cancelando sus membresías de tarjeta de crédito mucho mas, que antes. Y descubre que habido un aumento de transacciones fraudulentas con tarjetas de crédito.

El problema es identificar las transacciones fraudulentas con tarjetas de créditos para poder detener la transacción antes de que se procese.

El objetivó es reducir el numero de clientes que finalizan su membresía a causa del fraude.

Una vez definido a esto debemos pasar las datos cualitativos a cuantitativos para tener una idea de lo que pasa. El éxito de este problema seria una reducción del 10% en el numero de clientes que presentan reclamaciones por transacciones fraudulentas en un periodo de 6 meses.


Ahora hay que pensar como pasar esto a un modelo de machine learning, ¿Qué resultados quiere obtener de su modelo?, debe ser algo especifico, algo que el modelo puede producir. En el ejemplo de la tarjeta de crédito fraudulenta el modelo debe determinar si la tarjeta es fraudulenta o no, por lo que es un modelo de clasificación binaria.

Origen de datos

Una vez definido problema empresarial, se debe definir que tipos de datos son útiles para alcanzar el objetivo esperados, si se tiene acceso a esos datos, si se cuenta con una gran numero de datos.

Se debe resolver estos problemas ya que los datos son la fuente importantes para entrenar nuestro modelo. y hay tres tipos de datos principales.

  • Datos privados
  • Datos comerciales
  • Datos de código abierto

Los principales problemas que se debe abordar cuando se selecciona los datos necesarios para entrenar nuestro modelo tenemos:

  • Datos necesarios para abordar el problema.
  • Cantidad de datos, y su origen.
  • Accesibilidad de los mismo.
  • Capacidad de centralizar repositorio.
  • Representativa de los datos
  • Calidad de los datos.

Tambien es importante identificar las características y etiquetas de los datos, una característica es un atributo que se puede utilizar para ayudar a identificar patrones y predecir respuestas futuras. Cuando los datos ya están identificados , se denominan datos etiquetados.

Ingeniería de características.

Se trata los datos para hacerlo útiles. Se eligen de los datos las características que serán usadas en los modelos para entrenarlos mejor. El modelo usa los valores característica de nuevos datos para predecir el objetivo.

Extracción y selección de características

Si se quiere mejorar el modelo siempre es bueno realizar este proceso.

La extracción es usar datos no procesados, para reformular, combinar y transformar características primarias en otras nuevas. Brindando un nuevo conjunto que el modelo puede utilizar para mejorar su predicciones.

La selección de características consiste en seleccionar las características mas relevantes y descarta el resto. Para evitar redundancia o irrelevancia de las características existentes o para obtener un numero limitado de características.

Extracción de características.

Proceso de transformación de los datos para poder usarlos, abarca muchas actividades, que van desde la administración de los datos que faltan hasta la conversión de datos de textos en datos numéricos. Los datos deben estar en el formato correcto. Puede combinar datos o eliminarlos, decidir como tratar con los valores atípicos y posiblemente reescalar los datos.

Las tareas mas comunes son:

  • Codificación de datos ordinales y no ordinales.
  • Búsqueda de datos faltantes.
  • Tratamiento de los valores atípicos en los datos

Selección de características.

Luego de extraer las características, se debe filtrar por aquellas características útiles:

  • Método de filtro: Utilizan métodos estadísticos para medir su relevancia de las características por su correlación con la variable objetivó.
  • Método encapsulador: Miden la utilidad de un subconjunto de la característica al entrenar un modelo con él y al medir el éxito del modelo.
  • Métodos incrustación: son específicos del algoritmo y pueden utilizar una combinación de ambos.

El método de filtro son mas rápidos y baratos debido a que no hay que entrenar varias veces el modelo, el encapsulador suele encontrar mejores subconjuntos de las características.

Preparación de datos.

Los datos que se usen para entrenar el modelo, pueden estar sucios, ya sean datos duplicados o de una población incorrecta para el caso que se entrena el modelo. Por eso Existen cuatro consideraciones importantes:

  • Codificación de datos
  • Limpieza de datos
  • Búsqueda de datos faltantes
  • Tratamiento de los valores atípicos

Entrenamiento de modelos

Luego de la limpieza de los datos, se usaran los mismo para entrenarlos pero no se usan todos, si no que debemos guardar un porcentaje para probar el algoritmo.

Los datos de entrenamiento alimentan al modelo, y luego usamos el conjunto de validación para comprobar sus predicciones.

El conjunto de validación es donde puede notar cosas que querrá retocar, ajustar y cambiar.

El conjunto de datos de prueba solo incluye características, se desea predecir las etiquetas. El rendimiento de los resultado a aqui se asemeja al si lo tuviéramos en producción.

Sobreajuste e infraajuste

Cuando se tiene el modelo se busca que generalice bien. Quiere que el modelo funcione bien tanto con datos ya probados y con datos que no se tienen. Por eso debemos conocer que sucede cuando se sobreajuste o infraajuste el modelo. Y he de ahi la importancia de dividir los datos.

  • Sobreajuste
  • Infraajuste
  • Equilibrio

Evaluación y ajuste

Luego re resolver el tipo de modelo que se usara y ajustado. Se debe evaluar sus predicciones con datos nuevos y futuros.

Se evalúa el modelo con los datos de validación

Los datos de prueba comprueban la precisión del modelo, si el modelo funciona bien con los datos de prueba es probable que funcione bien con los nuevos datos.

Métrica de éxito.

En la formulación del problema se estableció una métrica o una medida para saber si se tuvo éxito en resolver el problema. La métrica que se use para medir el éxito del modelo debe estar relacionada con esa métrica.

Ajuste

Puede utilizar una instancia del modelo que no haya predicho para realizar una predicción. Ya conoces los datos de prueba puede comparar en sus objetivos de prueba. Cambiar las características o los hiperparámetros del modelo que ayuden a mejorar sus resultados.

Implementación del modelo

Luego de haber entrenado, ajustado y probado el modelo. Se debe realizar métricas de rendimiento cuando se implementa en producción, ya que no es lo mismo implementarla en en pruebas que en producción.
