S3 (Simple Storage Service)

Servicio de Amazon Web Services(AWS), con el propósito de almacenar archivos estáticos de cualquier tipo(imágenes, .js, etc.). Los almacena en Buckets o objectos.

Existen servicios que son globales y afectan a todas las regiones.

Pero en S3 el Bucket es nivel a región, por lo que tendrás que elegir bien la región. Otra característica de un bucket es que los nombres debe ser únicos a nivel mundial.

Configuraciones.

En la configuración de propiedades del objecto te pregunta si deseas que los objectos puedan ser cambiados por usuarios de AWS. Lo recomendable es que no.

Se puede configurar para que el servicio sea publico y accesible por todos pero eso no seria una buena practica ya que AWS tiene un servicio para ello llamado CloudFront. Que se encargaría del acceso a S3

Cifrado

S3 permite generar un cifrado a la información que guardemos en nuestros objectos.

El cifrado es importante porque si no se cifra permite que sea mas fácil leer logs o información importante para los atacantes

Clases de almacenamiento.

Existen diferentes tipos de almacenamiento que dependiendo de que tanto necesites acceder a esos ficheros y latencia del mismo podría variar en ello. Mientras mas rápido y accesible es mas caro se vuelve pero mientras mas lento y difícil de acceder mas barato. Evaluar que tipo de almacenamiento deberías dependerá de esos dos factores. Velocidad y disponibilidad.

Amazon S3 ofrece diferentes clases de almacenamiento, cada una diseñada para diferentes necesidades de acceso, disponibilidad y costos. A continuación, te detallo los tipos de almacenamiento en S3, sus características y una comparación de precios estimados (los precios pueden variar según la región y el uso específico).

1. S3 Standard

Uso: Datos que se acceden con frecuencia (sitios web, aplicaciones en tiempo real, backups activos).
Disponibilidad: 99.99%
Durabilidad: 99.999999999% (11 nueves)
Costo estimado: ~$0.023 por GB/mes
Retribución por acceso: No hay costos adicionales por acceso.

Cuándo usarlo:
Ideal para almacenar datos críticos a los que necesitas acceso frecuente y rápido.

2. S3 Standard - Infrequent Access (IA)

Uso: Datos a los que accedes ocasionalmente, pero que aún necesitas disponibles con baja latencia (ej. backups a mediano plazo, almacenamiento de registros).
Disponibilidad: 99.9%
Durabilidad: 99.999999999%
Costo estimado: ~$0.0125 por GB/mes
Costo de recuperación: ~$0.01 por GB recuperado

Cuándo usarlo:
Ideal para archivos que no necesitas con frecuencia, pero que aún quieres disponibles sin demoras.

3. S3 One Zone - Infrequent Access

Uso: Datos que no cambian con frecuencia y pueden almacenarse en una única zona de disponibilidad (ej. backups secundarios).
Disponibilidad: 99.5% (menor que otras clases porque está en una sola zona de AWS)
Durabilidad: 99.999999999%
Costo estimado: ~$0.01 por GB/mes
Costo de recuperación: ~$0.01 por GB

Cuándo usarlo:
Si puedes tolerar una posible pérdida de datos en caso de fallo en la única zona donde se almacena.

4. S3 Glacier

Uso: Archivos que necesitas almacenar por largos períodos sin acceso frecuente (ej. archivos históricos, backups de cumplimiento).
Disponibilidad: 99.99%
Durabilidad: 99.999999999%
Costo estimado: ~$0.004 por GB/mes
Costo de recuperación: Desde $0.01 a $0.03 por GB, según la velocidad de recuperación elegida.

Cuándo usarlo:
Si necesitas un almacenamiento de bajo costo para datos que no accederás por meses o años.

5. S3 Glacier Deep Archive

Uso: Almacenamiento a muy largo plazo (ej. archivos que deben guardarse durante décadas por regulaciones legales).
Disponibilidad: 99.99%
Durabilidad: 99.999999999%
Costo estimado: ~$0.00099 por GB/mes
Costo de recuperación: Desde $0.02 por GB (tarda entre 12 a 48 horas).

Cuándo usarlo:
Si necesitas guardar datos por años y el acceso puede esperar varias horas.

6. S3 Intelligent-Tiering

Uso: Alterna entre S3 Standard y S3 IA automáticamente según la frecuencia de acceso.
Disponibilidad: 99.9%
Durabilidad: 99.999999999%
Costo estimado: ~$0.023 por GB/mes en acceso frecuente y ~$0.0125 en IA
Costo de monitoreo: ~$0.0025 por 1,000 objetos por mes.

Cuándo usarlo:
Si no sabes con certeza cuándo accederás a los datos y quieres optimizar costos automáticamente.

Permisos / Políticas de Permisos.

En los Buckets necesitamos políticas para acceder a los objectos almacenados.

En la documentación de Amazo Web Services tiene ejemplos de políticas, que se pueden usar para iniciar proyectos, pero he de recalcar que cuenta con muchas configuraciones que controlan diferentes aspectos del bucket. Normalmente es un JSON que contiene nuestra configuración:

  • Version: la versión de nuestra configuración.
  • Statement(declaraciones): Contiene una lista de reglas que definen los permisos
  • Effect: Define si la ó esta permitida o denegada.
  • Principal: Define quien puede acceder al recurso, si se usa "*" es accesible por todo el mundo o público.
  • Action: la acción que puede hacer el usuario o las que no puede hacer:
    • "s3:GetObject":Leer un objeto
    • "s3:PutObject": Subir un objeto
    • "s3:DeleteObject": Eliminar un objeto
    • "s3:ListBucket": Listar los archivos en un bucket
  • Resource: Especifica que objectos u bucket aplica la política.

Ejemplo de bucket púplico:

{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Principal": "*",
      "Action": "s3:GetObject",
      "Resource": "arn:aws:s3:::mi-bucket/*"
    }
  ]
}

Ejemplo de bucket restringido:

{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Principal": { "AWS": "arn:aws:iam::123456789012:user/mi-usuario" },
      "Action": [
        "s3:PutObject",
        "s3:DeleteObject"
      ],
      "Resource": "arn:aws:s3:::mi-bucket/*"
    }
  ]
}

Permisos para denegar todo:

{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Deny",
      "Principal": "*",
      "Action": "s3:*",
      "Resource": "arn:aws:s3:::mi-bucket/*",
      "Condition": {
        "Bool": { "aws:SecureTransport": "false" }
      }
    }
  ]
}

Tambien se pueden agregar condiciones .
