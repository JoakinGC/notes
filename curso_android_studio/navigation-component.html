Navigation Component

Navigation Component en Android

El Navigation Component es la forma recomendada para manejar la navegación entre pantallas y Fragments en una aplicación Android. Permite una gestión más sencilla y estructurada de las transiciones entre pantallas, incluso en aplicaciones con múltiples fragmentos.

Configuración de Navigation Component

Para usar Navigation Component, primero debemos agregar sus dependencias en build.gradle.kts:

val version = "2.7.5"  

// Implementación de Navigation
implementation("androidx.navigation:navigation-fragment-ktx:$version")  
implementation("androidx.navigation:navigation-ui-ktx:$version")

Luego, sincronizamos el proyecto en Android Studio y creamos los fragmentos que vayamos a utilizar.

Navigation Graph

El Navigation Graph es una herramienta visual que muestra todas las pantallas y fragmentos de la aplicación en un mapa interactivo, facilitando la navegación entre ellos.

Creación del Navigation Graph

  1. En la vista de Android Studio, hacer clic derecho en res/ → New → Navigation Resource File.
  2. Asignarle un nombre, por ejemplo: nav_graph.xml.
  3. Agregar los fragmentos al Navigation Graph.

Esto nos permite organizar y visualizar fácilmente la navegación entre pantallas.

En el modo visual, podemos arrastrar fragmentos y establecer conexiones entre ellos con flechas. También es posible agrupar fragmentos para mejorar la organización.

Código XML del Navigation Graph

Internamente, cada pantalla o Fragment agregado al Navigation Graph se define en un archivo XML.

<?xml version="1.0" encoding="utf-8"?>  
<navigation xmlns:android="http://schemas.android.com/apk/res/android"  
    xmlns:app="http://schemas.android.com/apk/res-auto"  
    xmlns:tools="http://schemas.android.com/tools"  
    android:id="@+id/main_graph"  
    app:startDestination="@id/mainActivity">  

    <fragment  
        android:id="@+id/firstFragment"  
        android:name="com.practica.navigationcomponentexample.FirstFragment"  
        android:label="fragment_first"  
        tools:layout="@layout/fragment_first" />
</navigation>
  • app:startDestination="@id/mainActivity" → Define la pantalla principal de la aplicación.
  • <fragment> → Cada pantalla o fragmento se declara con su id y la clase correspondiente.

Agregar Navigation en una Activity

Los Fragment no pueden existir por sí solos; necesitan una Activity para contenerlos. Para esto, en el XML de la Activity, agregamos un FragmentContainerView, que alojará los fragmentos y manejará la navegación.

<?xml version="1.0" encoding="utf-8"?>  
<androidx.constraintlayout.widget.ConstraintLayout  
    xmlns:android="http://schemas.android.com/apk/res/android"  
    xmlns:app="http://schemas.android.com/apk/res-auto"  
    xmlns:tools="http://schemas.android.com/tools"  
    android:layout_width="match_parent"  
    android:layout_height="match_parent"  
    tools:context=".MainActivity">  

    <androidx.fragment.app.FragmentContainerView  
        android:id="@+id/navHostFragment"  
        android:name="androidx.navigation.fragment.NavHostFragment"  
        android:layout_width="0dp"  
        android:layout_height="0dp"  
        app:layout_constraintTop_toTopOf="parent"  
        app:layout_constraintEnd_toEndOf="parent"  
        app:layout_constraintStart_toStartOf="parent"  
        app:layout_constraintBottom_toBottomOf="parent"  
        app:navGraph="@navigation/nav_graph"  
        app:defaultNavHost="true"/>  

</androidx.constraintlayout.widget.ConstraintLayout>

Explicación

  • android:name="androidx.navigation.fragment.NavHostFragment" → Indica que este contenedor gestionará los fragmentos.
  • app:navGraph="@navigation/nav_graph" → Especifica el archivo Navigation Graph que hemos creado.
  • app:defaultNavHost="true" → Define este FragmentContainerView como el principal en la aplicación.

Configurar navegación entre Fragmentos

Desde la interfaz visual de Navigation Graph, podemos definir transiciones con flechas o configurar la navegación manualmente en XML.

<fragment  
    android:id="@+id/firstFragment"  
    android:name="com.practica.navigationcomponentexample.FirstFragment"  
    android:label="fragment_first"  
    tools:layout="@layout/fragment_first">  

    <action  
        android:id="@+id/action_firstFragment_to_secondFragment"  
        app:destination="@id/secondFragment"/>
</fragment>

Esto indica que, al ejecutar la acción action_firstFragment_to_secondFragment, se navegará a secondFragment.

Podemos también definir animaciones en la transición entre fragmentos:

<action  
    android:id="@+id/action_firstFragment_to_secondFragment"  
    app:destination="@id/secondFragment"  
    app:enterAnim="@anim/nav_default_enter_anim"  
    app:exitAnim="@anim/nav_default_exit_anim"  
    app:popEnterAnim="@anim/nav_default_pop_enter_anim"  
    app:popExitAnim="@anim/nav_default_pop_exit_anim"/>

Las animaciones pueden ser personalizadas en la carpeta res/anim.

Navegar entre Fragmentos en Kotlin

Podemos movernos entre fragmentos con findNavController() y la acción definida en el XML:

findNavController().navigate(R.id.action_firstFragment_to_secondFragment)

Si necesitamos pasar datos entre fragmentos, la mejor práctica es usar Safe Args.

Safe Args: Pasar Datos de Forma Segura

Safe Args es una dependencia recomendada por Google para transferir datos entre fragmentos sin riesgo de errores.

Agregar Safe Args en build.gradle.kts

id("androidx.navigation.safeargs.kotlin") version "2.7.1" apply false

Luego, sincronizamos el proyecto.

Definir Argumentos en el Navigation Graph

Podemos agregar argumentos en la interfaz visual o manualmente en XML:

<fragment  
    android:id="@+id/secondFragment"  
    android:name="com.practica.navigationcomponentexample.SecondFragment"  
    android:label="fragment_second"  
    tools:layout="@layout/fragment_second">  

    <action  
        android:id="@+id/action_secondFragment_to_thirdFragment"  
        app:destination="@id/thirdFragment"/>  

    <argument  
        android:name="name"  
        app:argType="string"/>
</fragment>

Esto define un argumento llamado name de tipo String.

Pasar Argumentos al Fragmento

En el FirstFragment, usamos Safe Args para enviar el dato:

findNavController().navigate(
    FirstFragmentDirections.actionFirstFragmentToSecondFragment(
        name = "Pepe"
    )
)

¿Qué hace FirstFragmentDirections?
Kotlin genera automáticamente una clase con los métodos adecuados para cada acción de navegación, evitando errores al pasar argumentos.

Recibir Argumentos en el Segundo Fragmento

class SecondFragment : Fragment() {
    override fun onCreateView(
        inflater: LayoutInflater, container: ViewGroup?,
        savedInstanceState: Bundle?
    ): View? {
        val binding = FragmentSecondBinding.inflate(inflater, container, false)
        
        val args = SecondFragmentArgs.fromBundle(requireArguments())
        binding.textView.text = args.name // Muestra el valor recibido

        return binding.root
    }
}

Conclusión

El Navigation Component facilita la gestión de pantallas en una aplicación, permitiendo navegar entre Activitys y Fragments de manera sencilla.

  • Navigation Graph organiza visualmente las transiciones.
  • Safe Args permite pasar datos de forma segura entre fragmentos.
  • NavHostFragment actúa como contenedor principal de los fragmentos.
