Estructura de un Proyecto en Android Studio

Siempre trabajamos con la vista Android, ya que es la más sencilla y organizada.

Al crear una aplicación, se divide en dos partes:

  1. La parte visual, que define la interfaz del usuario.
  2. La parte lógica, que es la que interactúa con la interfaz y procesa la información.

Una Activity es la unión entre la parte lógica y visual.

Cuando creamos una nueva Activity (New > Activity > Empty), Android Studio solicita un nombre para la Activity y también un nombre para el archivo XML correspondiente dentro de xml].

En este proceso, aparece una opción llamada "Launch app", donde indicamos cuál será la pantalla principal de la aplicación.

Clases y el ciclo de vida de una Activity

Una clase es un contenedor de Funciónes (aunque puede incluir más elementos).

onCreate() es el método que siempre se ejecuta al iniciar una pantalla en la aplicación:

override fun onCreate(savedInstanceState: Bundle?) {  
    super.onCreate(savedInstanceState)  
    setContentView(R.layout.activity_first_app)  
    // Se ejecuta al iniciar la pantalla  
}

Cada Activity tiene su parte lógica y un diseño XML asociado. setContentView(R.layout.activity_first_app) es la función que carga el diseño desde el XML correspondiente.

El archivo XML de cada Activity siempre se encuentra en res/layout, dentro de xml.

Configuración del Emulador

Podemos configurar el emulador de Android desde la esquina superior de Android Studio.
Aquí podemos definir:

  • La cantidad de RAM asignada.
  • El espacio de almacenamiento interno.
  • La versión de Android y el modelo del dispositivo.
  • Si incluye los Google Services (Maps, Google, YouTube, etc.).

Android Manifest

El proyecto contiene el archivo Android Manifest, que es clave para definir configuraciones globales de la aplicación.

Acceso a Componentes del XML desde la Activity

Desde una Activity podemos acceder a cualquier componente del archivo XML dentro de xml.

Esto se puede hacer dentro de la función onCreate() o en onResume().

Ejemplo de acceso a un componente

package com.example.estudioandroid.firstapp  

import androidx.appcompat.app.AppCompatActivity  
import android.os.Bundle  
import com.example.estudioandroid.R  

class FirstAppActivity : AppCompatActivity() {  
    override fun onCreate(savedInstanceState: Bundle?) {  
        super.onCreate(savedInstanceState)  
        setContentView(R.layout.activity_first_app)  

        val btnStart = findViewById<TipoDeComponente>(R.id.IDdeElemento) 
        // Tipo de componente puede ser TextView, Button, etc.
    }  
}

Ejemplo de agregar un Listener para un botón

package com.example.estudioandroid.firstapp  

import androidx.appcompat.app.AppCompatActivity  
import android.os.Bundle  
import android.util.Log  
import androidx.appcompat.widget.AppCompatButton  
import com.example.estudioandroid.R  

class FirstAppActivity : AppCompatActivity() {  
    override fun onCreate(savedInstanceState: Bundle?) {  
        super.onCreate(savedInstanceState)  
        setContentView(R.layout.activity_first_app)  

        val btnStart = findViewById<AppCompatButton>(R.id.btnStart)  

        btnStart.setOnClickListener {  
            // Código de acción  
            Log.i("Joaquin", "Mensaje")  
        }  
    }  
}

Ejemplo de obtener el contenido de un editText

package com.example.estudioandroid.firstapp  

import androidx.appcompat.app.AppCompatActivity  
import android.os.Bundle  
import android.util.Log  
import androidx.appcompat.widget.AppCompatButton  
import androidx.appcompat.widget.AppCompatEditText  
import com.example.estudioandroid.R  

class FirstAppActivity : AppCompatActivity() {  
    override fun onCreate(savedInstanceState: Bundle?) {  
        super.onCreate(savedInstanceState)  
        setContentView(R.layout.activity_first_app)  

        val btnStart = findViewById<AppCompatButton>(R.id.btnStart)  
        val etName = findViewById<AppCompatEditText>(R.id.etName)  

        val userInput = etName.text.toString() // Obtiene el contenido del EditText  

        btnStart.setOnClickListener {  
            Log.i("Joaquin", "Mensaje")  
        }  
    }  
}

Estructura de la Carpeta res/

Dentro de la carpeta res/ se encuentra la parte visual de la aplicación.

  • res/values/ → Contiene los colores y textos predefinidos.
    • En colors.xml podemos definir todos los colores a utilizar en el diseño de la aplicación.
    • En strings.xml se colocan todos los textos utilizados en los TextView, evitando escribirlos directamente en el código. Esto permite la adaptación a múltiples idiomas.
  • res/drawable/ → Almacena todas las imágenes utilizadas en la aplicación.
    • Para agregar nuevas imágenes, deben colocarse dentro de esta carpeta.

Atajos y Herramientas Útiles

  • Formatear código → Ctrl + Alt + L
  • Verificar la versión de Android Studio → Ir a Help > About

Control de Estilos y Modo Oscuro

Dentro de res/values/themes/ se encuentran dos archivos XML:

  1. Un tema claro
  2. Un tema oscuro

Para personalizar los estilos de la aplicación, se pueden modificar estos archivos o crear un nuevo archivo en styles.xml. Esto permite cambiar colores y personalizar la apariencia de la aplicación según las necesidades del proyecto.
