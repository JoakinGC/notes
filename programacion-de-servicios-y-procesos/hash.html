Hash en Seguridad Informática

El hash es una función criptográfica que transforma una entrada de datos en una cadena de longitud fija, conocida como digest. Es ampliamente utilizado en seguridad informática para almacenar contraseñas, verificar integridad de datos y en criptografía.

Características de una Función Hash Segura

Para ser considerada segura, una función hash debe cumplir con:

  1. Determinismo: La misma entrada siempre produce el mismo hash.
  2. Eficiencia: El cálculo del hash debe ser rápido.
  3. Preimagen resistente: No debe ser posible obtener la entrada original a partir del hash.
  4. Resistencia a colisiones: Dos entradas diferentes no deben generar el mismo hash.
  5. Avalancha: Un pequeño cambio en la entrada debe producir un hash completamente diferente.

Algoritmos de Hash más Usados

Algoritmo Longitud del Hash Seguridad
MD5 128 bits Inseguro (colisiones)
SHA-1 160 bits Inseguro (colisiones)
SHA-256 256 bits Seguro
SHA-512 512 bits Seguro
Bcrypt Variable Seguro (usado en contraseñas)

Ejemplo de Hash en Python

import hashlib

texto = "Seguridad Informática"
hash_sha256 = hashlib.sha256(texto.encode()).hexdigest()
print("SHA-256 Hash:", hash_sha256)

Uso de Hash en Seguridad

1. Almacenamiento de Contraseñas

Las contraseñas no deben almacenarse en texto plano. Se usa hashing con sal (salted hashing) para mayor seguridad.

Ejemplo con Bcrypt en Python:

import bcrypt

password = "mi_secreta_contraseña".encode()
salt = bcrypt.gensalt()
hashed_password = bcrypt.hashpw(password, salt)
print("Hash almacenado:", hashed_password)

2. Verificación de Integridad de Datos

Se usa hashing para comprobar que un archivo no ha sido alterado.

import hashlib

def calcular_hash(archivo):
    hasher = hashlib.sha256()
    with open(archivo, 'rb') as f:
        while chunk := f.read(4096):
            hasher.update(chunk)
    return hasher.hexdigest()

print("Hash del archivo:", calcular_hash("archivo.txt"))

3. Firmas Digitales

Las funciones hash son usadas en criptografía para garantizar la autenticidad de los mensajes y firmas digitales.

Ataques Comunes a Funciones Hash

  1. Ataques de Preimagen: Intento de encontrar una entrada que produzca un hash específico.
  2. Ataques de Colisión: Buscar dos entradas diferentes que produzcan el mismo hash.
  3. Ataques de Diccionario y Fuerza Bruta: Usados para descifrar contraseñas hash almacenadas sin protección adecuada.

Conclusión

Las funciones hash son esenciales en seguridad informática, pero deben usarse correctamente. Algoritmos antiguos como MD5 y SHA-1 han sido comprometidos y deben evitarse. Para almacenamiento seguro de contraseñas, Bcrypt es una mejor opción, y para integridad de datos, SHA-256 sigue siendo una opción confiable.
