Delegación de los Eventos

No es optimo utilizar  tantos addEventListener()

Ni tampoco forEach() o un bucle tampoco es optimo

La forma de hacerlo con vanilla JS es con evento padre que desencadene los eventos asociados a ese

Si es un formulario tenemos que validar treinta datos no vamos a generar un evento para cada validación lo que se va a hacer es tratar al evento formulario como padre y cuando termine de escribir todo ahi realizar las validaciones es mas se puede colocar como elemento padre el document.

document.addEventListener("click",(e) =>{

   console.log("Click en", e.target);

})

De esta manera nosotros asignamos el evento padre que controla el flujo de los siguientes eventos.

e.target es la propiedad que dice que tipo etiqueta empezo el evento entonces con eso. Nosotros hacermos un flujo de control con if y el método match o includes y analizamos si el evento concide con la clase y etiqueta que nosotros colocamos.

document.addEventListener("click",(e) =>{

   console.log("Click en", e.target);




   if(e.target.matches(".eventos-flujo div")){

       flujoDeEventos(e);

   }



   if(e.target.matches(".eventos-flujo a")){

       alert("Hola soy tú amigo y docente digital... Jonathan Mircha");

       a.preventDefault();

   }

})
