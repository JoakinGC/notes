WeakSets & WeakMaps

Son variantes de los Seat y los Maps

Son variantes debiles de estos datos ya que cuando nuestro código o flujo de trabajo deja de usar estos dato JS los elimina en un determinado tiempo que depende del navegador y eso hace que sea una mejora de rendimiento.

Igual de presentar ventajas tambien tiene algunas desventajas.

No se pueden iterar sobre ellos o recorrer en ellos. No son elementos iterables.

Tampoco se puede usar el método .clear() de los sets.

Tampoco tienen la propiedad size

Con los weakSet no puede definir sus elementos a la vez que los declaras si o si se usa el método add.()

A la hora de sumar elemetos los weakSet solo aceptan valores dentro de objectos no numericos, booleanos, etc.

weakSet.delete() elemina un elemento del weakSet que haya en el mismo.

Cuando los valores dentro de weakSet son null o undifined estos pasan a eliminarse automaticamente. O weakSet queda vacio sin una propiedad

Con los Weak Map si se usa set() para sumar elementos.

Los elemetosn dentro del Weak Map no son elementos iterables.

Al igual que weak set se puede usar el .has() para comprobar si tiene una llave y este devolvera un booleano.

Tambien se puede eliminar una lleva con .delete() en weak set.
